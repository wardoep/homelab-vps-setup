SSH Access Workflow

    Purpose
    This document covers how I use SSH to access and manage Linux-based systems in my personal setup.

    Systems I Access
    I use SSH across multiple parts of my environment, including:
    - Ubuntu server systems
    - VPS-based environments
    - Machines I access remotely through Tailscale

    What I Use SSH For
    - Logging into remote systems from the terminal
    - Managing files and navigating project directories
    - Running commands directly on Linux environments
    - Checking and maintaining assistant-related tools and services
    - Supporting my day-to-day workflow while learning more about remote system management

    What I've Learned
    Using SSH regularly has helped me get more comfortable with:
    - Remote Linux administration
    - CLI-based workflows
    - File and process management
    - Working across connected systems without needing a local GUI
