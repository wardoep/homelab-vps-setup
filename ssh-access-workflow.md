# SSH Access Workflow

## Purpose

This document covers how I use SSH to access and manage the Linux systems in my environment.

## Systems I Access

- Ubuntu Server systems
- VPS-based environments
- Machines reached remotely through Tailscale

## What I Use SSH For

- Logging into remote systems from the terminal
- Managing files and navigating project directories
- Running commands directly on remote Linux hosts
- Checking and maintaining tools and services running on the server
- Day-to-day administration without a local GUI

## What I've Learned

Using SSH daily has made me comfortable with:

- Remote Linux administration
- CLI-only workflows
- File and process management on remote hosts
- Working across several connected systems from a single terminal
